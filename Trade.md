# 간소화 무한매수법 실행 정리

이 문서는 기존 무한매수법 V4.0의 핵심 아이디어를 더 쉽게 실행하기 위해 단순화한 개인용 매매 규칙입니다.

원문 방법론과 완전히 동일한 방식은 아닙니다. 대신 다음 효과를 비슷하게 얻는 것을 목표로 합니다.

1. 원금을 여러 번에 나누어 매수한다.
2. 가격이 내려오면 정해진 금액만큼 추가 매수한다.
3. 반등하면 일부를 팔아 현금을 회수한다.
4. 현금이 거의 떨어지면 보유분 일부를 팔아 다시 매수 여력을 만든다.
5. 모든 주문은 사용자가 직접 증권사 앱에서 실행한다.

투자 판단과 손실 책임은 투자자 본인에게 있습니다. 이 문서는 실행 보조용 정리이며 투자 권유가 아닙니다.

## 1. 핵심 단순화

기존 방법론에서 어려운 부분을 다음처럼 단순화합니다.

| 기존 개념 | 간소화 버전 |
| --- | --- |
| T값 | 사용하지 않음 |
| 별지점 공식 | 사용하지 않음 |
| 전반전/후반전 | 구분하지 않음 |
| 소진후 리버스모드 | 회복모드로 단순화 |
| 복잡한 T 재계산 | 현금, 보유수량, 평단만 관리 |

매일 확인할 핵심 값은 4개입니다.

1. 현금
2. 보유수량
3. 평단
4. 최근 종가

## 2. 시작 설정

### 종목

1차 기준 종목은 `TQQQ`, `SOXL`만 사용합니다.

| 종목 | 성격 |
| --- | --- |
| TQQQ | 상대적으로 낮은 변동성, 초보자 우선 |
| SOXL | 더 높은 변동성, 더 공격적 |

### 원금과 분할 수

전략마다 원금을 정하고 분할 수를 선택합니다.

| 분할 수 | 성향 |
| --- | --- |
| 20분할 | 공격적 |
| 40분할 | 방어적 |

기본 1회 매수금은 다음과 같습니다.

```text
기본 1회 매수금 = 원금 / 분할 수
```

예시입니다.

```text
원금 20,000달러, 40분할
=> 기본 1회 매수금 = 500달러
```

## 3. 모드 구분

간소화 버전은 모드를 2개만 사용합니다.

| 모드 | 의미 |
| --- | --- |
| 일반모드 | 현금이 충분한 일반적인 분할매수 구간 |
| 회복모드 | 현금이 거의 부족해 일부 매도로 현금을 다시 만드는 구간 |

### 일반모드 조건

```text
현금 >= 기본 1회 매수금
```

현금이 기본 1회 매수금 이상이면 일반모드로 봅니다.

### 회복모드 진입 조건

```text
현금 < 기본 1회 매수금
그리고 보유수량 > 0
```

현금이 1회 매수금보다 적어지면 더 이상 정상적인 분할매수가 어렵다고 보고 회복모드로 전환합니다.

## 4. 일반모드 매수

일반모드에서는 하루에 최대 1회분만 매수합니다.

### 보유수량이 0일 때

첫 진입은 단순하게 1회분만 매수합니다.

```text
매수금 = 기본 1회 매수금
```

증권사 앱에서 직접 1회분 매수를 실행합니다.

LOC를 쓸 수 있다면 현재가보다 조금 높은 가격에 LOC 매수를 걸어 체결 가능성을 높일 수 있습니다.

### 보유수량이 있을 때

평단 이하로 내려오면 1회분을 추가 매수합니다.

```text
매수 기준가 = 평단
매수금 = min(기본 1회 매수금, 현재 현금)
매수수량 = floor(매수금 / 매수 기준가)
```

주문 예시입니다.

```text
평단: 50.00달러
기본 1회 매수금: 500달러

LOC 매수: 50.00달러, 10주
```

평단 이하에서만 추가 매수하므로 규칙이 단순하고, 무리하게 높은 가격을 계속 따라 사는 것을 줄입니다.

## 5. 일반모드 매도

일반모드에서는 반등 매도와 최종 목표 매도를 함께 봅니다.

### 반등 매도

평단보다 일정 비율 이상 오르면 보유수량의 일부를 매도합니다.

| 종목 | 반등 매도 기준 | 매도 수량 |
| --- | --- | --- |
| TQQQ | 평단 +5% | 보유수량의 25% |
| SOXL | 평단 +7% | 보유수량의 25% |

계산식입니다.

```text
TQQQ 반등 매도가 = 평단 × 1.05
SOXL 반등 매도가 = 평단 × 1.07
반등 매도수량 = floor(보유수량 × 0.25)
```

### 최종 목표 매도

큰 반등이 오면 나머지 수량을 목표 수익률에 매도합니다.

| 종목 | 최종 목표 매도 기준 |
| --- | --- |
| TQQQ | 평단 +15% |
| SOXL | 평단 +20% |

계산식입니다.

```text
TQQQ 최종 매도가 = 평단 × 1.15
SOXL 최종 매도가 = 평단 × 1.20
최종 매도수량 = 보유수량 - 반등 매도수량
```

주문 예시입니다.

```text
TQQQ 평단: 50.00달러
보유수량: 40주

LOC 매도 또는 지정가 매도: 52.50달러, 10주
지정가 매도: 57.50달러, 30주
```

## 6. 회복모드

회복모드는 기존 방법론의 리버스모드를 더 쉽게 바꾼 것입니다.

목표는 단순합니다.

1. 현금이 부족하면 보유분 일부를 팔아 현금을 만든다.
2. 최근 평균보다 내려오면 그 현금으로 다시 산다.
3. 가격이 어느 정도 회복되면 일반모드로 돌아간다.

### 회복모드 첫날

회복모드에 들어간 첫날은 매수하지 않고 일부만 매도합니다.

| 분할 수 | 매도 수량 |
| --- | --- |
| 20분할 | 보유수량의 10% |
| 40분할 | 보유수량의 5% |

계산식입니다.

```text
20분할 회복 매도수량 = floor(보유수량 × 0.10)
40분할 회복 매도수량 = floor(보유수량 × 0.05)
```

주문 방식은 단순하게 `MOC 매도` 또는 장중 직접 매도로 처리합니다.

### 회복모드 기준가

회복모드에서는 복잡한 별지점 대신 최근 5거래일 평균 종가만 사용합니다.

```text
회복 기준가 = 최근 5거래일 종가 평균
```

### 회복모드 둘째 날 이후

둘째 날부터는 회복 기준가를 중심으로 행동합니다.

| 조건 | 행동 |
| --- | --- |
| 현재가 또는 종가가 회복 기준가보다 높음 | 보유수량 일부 매도 |
| 현재가 또는 종가가 회복 기준가보다 낮음 | 현금의 25%로 매수 |

매도 수량은 첫날과 동일합니다.

```text
20분할: 보유수량의 10%
40분할: 보유수량의 5%
```

매수금은 다음과 같습니다.

```text
회복모드 매수금 = 현재 현금 × 0.25
매수수량 = floor(회복모드 매수금 / 회복 기준가)
```

이 규칙은 가격이 강할 때 조금 팔아 현금을 만들고, 약할 때 일부 현금으로 다시 사는 방식입니다.

## 7. 회복모드 종료

가격이 평단 대비 일정 수준 이상 회복하면 회복모드를 종료하고 일반모드로 돌아갑니다.

| 종목 | 일반모드 복귀 기준 |
| --- | --- |
| TQQQ | 종가가 평단 × 0.85보다 높음 |
| SOXL | 종가가 평단 × 0.80보다 높음 |

즉, 손실 구간이 너무 깊지 않은 수준까지 회복되면 다시 일반모드로 돌아갑니다.

```text
TQQQ 복귀 기준 = 평단 대비 -15%보다 위
SOXL 복귀 기준 = 평단 대비 -20%보다 위
```

## 8. 사이클 종료

보유수량이 0이 되면 한 사이클이 종료된 것으로 봅니다.

이후 선택지는 두 가지입니다.

| 방식 | 설명 |
| --- | --- |
| 단리 | 처음 정한 원금 기준으로 다시 시작 |
| 복리 | 수익금을 포함한 새 금액으로 다시 시작 |

개인용 앱에서는 사이클 종료 후 `새 사이클 시작` 버튼으로 원금과 잔금을 다시 설정할 수 있게 하면 됩니다.

## 9. 매일 실행 체크리스트

### 주문 전

1. 전략을 선택한다.
2. 현재 현금을 입력한다.
3. 현재 보유수량을 입력한다.
4. 현재 평단을 입력한다.
5. 최근 종가를 입력한다.
6. 회복모드라면 최근 5거래일 종가를 확인한다.
7. 앱에서 `오늘 주문 계산` 버튼을 누른다.

### 일반모드에서 앱이 알려줄 내용

1. 오늘 매수할 가격과 수량
2. 반등 매도 가격과 수량
3. 최종 목표 매도 가격과 수량
4. 현금이 부족해 회복모드로 넘어가야 하는지 여부

### 회복모드에서 앱이 알려줄 내용

1. 최근 5거래일 평균 기준가
2. 오늘 매수인지 매도인지
3. 매수한다면 현금의 25% 기준 수량
4. 매도한다면 20분할 10%, 40분할 5% 기준 수량
5. 일반모드 복귀 조건을 만족했는지 여부

### 체결 후

1. 실제 매수 체결 수량과 평균가를 입력한다.
2. 실제 매도 체결 수량과 평균가를 입력한다.
3. 증권사 기준 현금, 보유수량, 평단을 입력한다.
4. 앱에 저장한다.
5. 다음 날은 저장된 값에서 다시 계산한다.

## 10. 주문 방식 요약

| 상황 | 주문 방식 |
| --- | --- |
| 일반모드 첫 매수 | 직접 매수 또는 LOC 매수 |
| 일반모드 추가 매수 | 평단 기준 LOC 매수 |
| 일반모드 반등 매도 | LOC 매도 또는 지정가 매도 |
| 일반모드 최종 매도 | 지정가 매도 |
| 회복모드 첫날 매도 | MOC 매도 또는 직접 매도 |
| 회복모드 이후 매수/매도 | 최근 5일 평균 기준으로 직접 판단 |

## 11. 간단한 판단표

```text
보유수량 = 0
=> 새 사이클 시작 또는 종료

현금 >= 기본 1회 매수금
=> 일반모드

현금 < 기본 1회 매수금 그리고 보유수량 > 0
=> 회복모드

일반모드에서 현재가 <= 평단
=> 1회분 매수 고려

일반모드에서 현재가 >= 평단 × 반등매도 기준
=> 25% 매도 고려

일반모드에서 현재가 >= 평단 × 최종목표 기준
=> 나머지 매도 고려

회복모드에서 가격 > 최근 5일 평균
=> 일부 매도해서 현금 확보

회복모드에서 가격 < 최근 5일 평균
=> 현금의 25%로 매수

회복모드에서 TQQQ 종가 > 평단 × 0.85
=> 일반모드 복귀

회복모드에서 SOXL 종가 > 평단 × 0.80
=> 일반모드 복귀
```

## 12. 장점과 포기하는 것

### 장점

1. T값을 계산하지 않아도 된다.
2. 별지점 공식을 외울 필요가 없다.
3. 매일 입력해야 할 값이 적다.
4. 앱으로 구현하기 쉽다.
5. 체결 후 상태 관리가 단순하다.

### 포기하는 것

1. 원문 방법론과 완전히 동일한 결과는 나오지 않는다.
2. 전반전/후반전의 섬세한 매수 강도 조절은 사라진다.
3. T값 기반의 정교한 회차 관리는 하지 않는다.
4. 시장 상황에 따라 원문보다 공격적이거나 보수적으로 작동할 수 있다.

이 간소화 버전의 목적은 최고로 정교한 매매가 아니라, 매일 지속하기 쉬운 규칙을 만드는 것입니다.
